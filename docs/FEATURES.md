# Feature Engineering Documentation

This document describes all features generated by the `FeatureEngineer` class for electricity demand forecasting.

## Overview

The feature engineering pipeline generates approximately 70+ features from raw hourly electricity load data. Features are organized into the following groups:

1. **Lag Features** - Historical load values
2. **Rolling Statistics** - Aggregated statistics over time windows
3. **Calendar Features** - Time-based categorical features
4. **Fourier Features** - Cyclic patterns using trigonometric functions
5. **Holiday Features** - Holiday indicators
6. **Temperature Features** - Weather-related features (when available)

---

## 1. Lag Features

Lag features capture the autocorrelation structure of the time series.

| Feature | Description | Rationale |
|---------|-------------|-----------|
| `lag_1h` | Load value 1 hour ago | Short-term momentum |
| `lag_2h` | Load value 2 hours ago | Short-term trend |
| `lag_3h` | Load value 3 hours ago | Recent pattern |
| `lag_6h` | Load value 6 hours ago | Quarter-day pattern |
| `lag_12h` | Load value 12 hours ago | Half-day pattern |
| `lag_24h` | Load value 24 hours ago | Daily seasonality |
| `lag_48h` | Load value 48 hours ago | Two-day pattern |
| `lag_168h` | Load value 168 hours (1 week) ago | Weekly seasonality |
| `same_hour_prev_day` | Same hour yesterday | Daily pattern reference |
| `same_hour_prev_week` | Same hour last week | Weekly pattern reference |
| `diff_1h` | 1-hour difference | Short-term change rate |
| `diff_24h` | 24-hour difference | Daily change |

---

## 2. Rolling Statistics

Rolling statistics capture trends and volatility over different time windows.

### Rolling Windows: 6h, 12h, 24h, 48h, 168h

For each window size, we compute:

| Feature | Description | Rationale |
|---------|-------------|-----------|
| `rolling_mean_{N}h` | Mean load over past N hours | Trend indicator |
| `rolling_std_{N}h` | Standard deviation over past N hours | Volatility measure |
| `rolling_min_{N}h` | Minimum load over past N hours | Lower bound reference |
| `rolling_max_{N}h` | Maximum load over past N hours | Upper bound reference |

**Note**: All rolling features use `shift(1)` to prevent data leakage - they only use data available at prediction time.

---

## 3. Calendar Features

Calendar features capture regular temporal patterns in electricity demand.

| Feature | Description | Range | Rationale |
|---------|-------------|-------|-----------|
| `hour` | Hour of day | 0-23 | Daily load profile |
| `day_of_week` | Day of week | 0-6 (Mon-Sun) | Weekday vs weekend patterns |
| `day_of_month` | Day of month | 1-31 | Monthly billing cycles |
| `month` | Month of year | 1-12 | Seasonal patterns |
| `quarter` | Quarter of year | 1-4 | Seasonal grouping |
| `year` | Year | Integer | Long-term trends |
| `is_weekend` | Weekend indicator | 0/1 | Reduced commercial activity |

---

## 4. Fourier Features

Fourier features model cyclic patterns using sine and cosine transformations. This approach is more effective than categorical encoding for cyclical data because it preserves the continuous, periodic nature of time.

### Daily Cycle (24-hour period)

| Feature | Description |
|---------|-------------|
| `daily_sin_1`, `daily_cos_1` | Fundamental daily frequency |
| `daily_sin_2`, `daily_cos_2` | 2nd harmonic (12-hour pattern) |
| `daily_sin_3`, `daily_cos_3` | 3rd harmonic (8-hour pattern) |

### Weekly Cycle (168-hour period)

| Feature | Description |
|---------|-------------|
| `weekly_sin_1`, `weekly_cos_1` | Fundamental weekly frequency |
| `weekly_sin_2`, `weekly_cos_2` | 2nd harmonic (mid-week pattern) |

### Annual Cycle (365.25-day period)

| Feature | Description |
|---------|-------------|
| `annual_sin_1`, `annual_cos_1` | Fundamental annual frequency |
| `annual_sin_2`, `annual_cos_2` | 2nd harmonic (6-month pattern) |
| `annual_sin_3`, `annual_cos_3` | 3rd harmonic (4-month pattern) |
| `annual_sin_4`, `annual_cos_4` | 4th harmonic (quarterly pattern) |

---

## 5. Holiday Features

Holiday features capture the impact of public holidays on electricity demand (typically reduced commercial/industrial load).

| Feature | Description | Rationale |
|---------|-------------|-----------|
| `is_holiday` | US federal holiday indicator | Reduced commercial activity |
| `is_day_before_holiday` | Day before a holiday | Pre-holiday behavior |
| `is_day_after_holiday` | Day after a holiday | Post-holiday ramp-up |

Holidays included (US federal holidays):
- New Year's Day
- Martin Luther King Jr. Day
- Presidents' Day
- Memorial Day
- Independence Day
- Labor Day
- Columbus Day
- Veterans Day
- Thanksgiving
- Christmas Day

---

## 6. Temperature Features

Temperature features capture the relationship between weather and electricity demand. These are only generated when temperature data is available.

| Feature | Description | Rationale |
|---------|-------------|-----------|
| `temperature` | Current temperature (°F) | Direct weather impact |
| `temp_lag_1h` | Temperature 1 hour ago | Recent weather trend |
| `temp_lag_24h` | Temperature 24 hours ago | Daily weather comparison |
| `temp_rolling_mean_24h` | 24-hour rolling mean temp | Sustained weather patterns |
| `temp_rolling_std_24h` | 24-hour rolling std temp | Weather variability |
| `heating_degree` | max(0, 65 - temp) | Heating load indicator |
| `cooling_degree` | max(0, temp - 65) | Cooling load indicator |
| `heating_degree_sq` | Squared heating degree | Non-linear heating effect |
| `cooling_degree_sq` | Squared cooling degree | Non-linear cooling effect |

### Temperature-Load Relationship

Electricity demand has a U-shaped relationship with temperature:
- **Cold weather** (below 65°F): Increased heating load
- **Mild weather** (around 65°F): Minimal HVAC load
- **Hot weather** (above 65°F): Increased cooling load

The squared terms capture the non-linear increase in load at extreme temperatures.

## Configuration

Features can be customized using `FeatureConfig`:

```python
from forecast_service.features import FeatureEngineer, FeatureConfig

config = FeatureConfig(
    lag_hours=[1, 24, 168],  # Custom lag periods
    rolling_windows=[24, 168],  # Custom rolling windows
    fourier_daily_terms=2,  # Fewer daily harmonics
    include_holidays=True,
    include_temperature_features=True,
)

engineer = FeatureEngineer(config=config)
features = engineer.transform(df)
```

---

## Data Leakage Prevention

All features are designed to prevent data leakage:

1. **Lag features**: Use `shift()` to only access past values
2. **Rolling features**: Use `shift(1)` before rolling to exclude current observation
3. **Holiday features**: Based on calendar dates, known in advance
4. **Temperature features**: In production, would use forecasted temperature

---

## Feature Importance

Based on typical model training, the most important feature groups are:

1. **Recent lags** (`lag_1h`, `lag_24h`, `lag_168h`) - Strongest predictors
2. **Rolling means** - Capture current load regime
3. **Fourier daily/weekly** - Capture regular patterns
4. **Temperature features** - Critical during extreme weather
5. **Holiday indicators** - Important for specific days

## Usage Example

```python
from forecast_service.data import SyntheticDataGenerator
from forecast_service.features import FeatureEngineer

# Generate or load data
generator = SyntheticDataGenerator()
data = generator.generate()

# Create features
engineer = FeatureEngineer()
features = engineer.transform(data, target_col="load_mw")

# Get feature names
print(f"Generated {len(engineer.feature_names_)} features")
print(engineer.get_feature_groups())
```
